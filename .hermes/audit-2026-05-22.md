## Issues Found & Fixed

### HIGH — Profile Provider Breakage (FIXED)
**Problem:** code-writer profile primary model `stepfun/step-3.5-flash` via `nous` returning
401/404 (no credits). Fallback providers opencode-zen and opencode-go rate-limited (429).
Workers exhaust all retries and exit cleanly — protocol violation crash loop.

**Fix:** Switched all three kanban profiles (code-writer, overseer, visual-reviewer) to
`gpt-5.4-mini` via `openai-codex` as primary. Code-writer fallbacks: nous → opencode-zen → opencode-go.

### MEDIUM — Print Without Flush (FIXED)
**Problem:** 2 print() calls in bridge-host.py __main__ block missing flush=True.
**Fix:** Added flush=True to both.

### MEDIUM — Duplicate Import (FIXED)
**Problem:** `urllib.request` imported at module level AND inside `_find_chatgpt_tab_cdp()`.
**Fix:** Removed inner import, used module-level reference.

### MEDIUM — __pycache__ Blocking Extension Load (FIXED)
**Problem:** Chrome rejects extension dirs containing `__pycache__`. Recompiled after edits.
**Fix:** Added cleanup at bridge-host.py startup entry point.

### LOW — Long Lines (>120 chars)
~6 lines in bridge-host.py exceed 120 chars. Cosmetic, reduces terminal readability.

### LOW — Deicide Redundant Files
An older bridge implementation and an unused initialization module were present and are candidates for removal in cleanup passes.
Consider removing in a cleanup pass.
